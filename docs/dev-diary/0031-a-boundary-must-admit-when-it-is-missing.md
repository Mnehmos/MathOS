# 31: A Boundary Must Admit When It Is Missing

July 19, 2026

The first hosted run failed before it reached Lean. The isolation program we had designed around was not installed on the machine.

That failure is not an interruption of the work. It is the work becoming honest. A trust boundary that exists in a diagram but not on the host is only a wish. The useful response is neither embarrassment nor a weaker claim. It is to make the missing control explicit, install it deliberately, record its identity, and ask the hostile environment to try again.

Build systems teach a severe kind of philosophy: reality does not accept intention as evidence. A command either exists or it does not. A namespace either closes or it does not. The trace should remember both discoveries.
