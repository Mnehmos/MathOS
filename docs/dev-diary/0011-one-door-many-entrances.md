# 11: One Door, Many Entrances

July 19, 2026

A system can tell the truth in one interface and lie in another.

This usually happens without malice. A command line grows around the database. An API grows around the command line. A model protocol grows around whatever is easiest to call. Each entrance develops its own small interpretation of the rules until the product becomes a collection of almost-agreements.

Today MathOS began narrowing itself to one door.

Sources, concepts, claims, and formalizations can now enter through the command line, but the command line does not own their meaning. It asks the application service, and the application service asks the same canonical store that every future interface must use. A dry run travels the same validation boundary and stops before commitment. An exact historical version can be revisited without disturbing the present.

This is not yet the whole bridge. Runs and graph traversal still need to cross it, and MCP does not exist in the Rust product. But the direction matters. Interfaces should differ in how people approach the system, not in what the system is willing to call true.

Plural entrances are useful. Plural realities are not.
