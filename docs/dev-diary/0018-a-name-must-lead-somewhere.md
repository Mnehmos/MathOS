# 18: A Name Must Lead Somewhere

July 19, 2026

A hash can create the feeling of certainty. Sixty-four hexadecimal characters look exact, and exactness is easy to mistake for existence.

Today MathOS stopped accepting that appearance. An environment hash may be well formed and still name nothing. Before a formalization can depend on it, the hash must lead back to a registered manifest whose contents recompute to the same identity. If the stored world changes, the name no longer fits. If the world was never stored, the name is only a claim about a missing object.

Registration gives the identity a history. It records who introduced the environment, refuses accidental repetition, survives restart, and cannot be silently rewritten or deleted. A dry run can predict the name without creating it. A later reader can recover the manifest without relying on the machine that first registered it.

This is a small lesson with a wide reach. Provenance is not decoration attached to an identifier. Provenance is the path by which an identifier resolves into something inspectable.

MathOS will carry many hashes: claims, artifacts, environments, events, releases. None should become a talisman. Every important name must lead somewhere.
